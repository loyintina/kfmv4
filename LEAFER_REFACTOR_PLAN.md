# KFM v4 LeaferJS 重构计划

> 目标：保持现有视觉风格，逐步用 LeaferJS 替换 DOM 渲染，实现设计工具级别的文件树交互体验。

---

## 核心原则

```yaml
风格不变:
  - 紫色主色调保持 (#7c3aed)
  - 左侧加粗边框风格保持
  - 网格背景保持
  - 光球面板交互保持
  - 光标高亮围框保持

渐进重构:
  - 不一次性推翻现有代码
  - 每个阶段可独立验证
  - 出问题可快速回退

性能优先:
  - 先解决性能瓶颈（滚动/渲染）
  - 后添加视觉效果（动画/连线）
```

## Phase 1：基础渲染层

**目标**：用 Leafer Canvas 替换 DOM 节点渲染，保持现有外观。

### 任务清单

1. **引入 LeaferJS**
   ```bash
   npm install leafer-ui
   ```

2. **创建 Leafer 容器**
   - 在 `.sidebar-content` 内创建 Leafer Canvas
   - Canvas 覆盖整个侧栏区域
   - 设置 `view: window` 或自定义容器

3. **节点渲染迁移**
   - 将现有 `.tree-item` DOM 元素转换为 Leafer 元素
   - 使用 `Rect` + `Text` 组合
   - 保持现有样式：
     - 左侧加粗边框 `border-left: 3px solid rgba(124,58,237,.3)`
     - 选中态边框 `rgba(0,212,255,.7)`
     - 文字颜色 `var(--text)`

4. **滚动适配**
   - Leafer 自带的视口滚动
   - 或保持现有 `.sidebar-content` 滚动，Leafer 只渲染可视区域

5. **交互映射**
   - Leafer 的 `draggable` 替换现有拖拽
   - Leafer 的 `pointer` 事件替换现有 click/touch

### 验证标准

- 文件树外观与现在一致
- 滚动流畅，无卡顿
- 点击/选择功能正常
- 光球面板交互不受影响

### 风险

- Leafer 与现有 CSS 样式可能冲突
- 滚动事件需要重新适配
- 光标位置计算需要从 DOM 坐标转 Leafer 坐标

### 回退方案

保留现有 DOM 树代码，通过开关控制使用 DOM 或 Leafer：
```javascript
const USE_LEAFER = false; // 出问题时改为 false
```

## Phase 2：动画增强层

**目标**：利用 Leafer 动画系统，实现流畅的展开/收起/光标动画。

### 任务清单

1. **目录展开动画**
   - 子节点从父节点位置弹性展开
   - 使用 Leafer `animate` + `spring` 缓动
   - 带延迟的瀑布式动画
   - 收起时反向收缩

2. **光标移动轨迹**
   - 光标切换时留下渐隐轨迹线
   - 使用 Leafer `Path` + `动画循环`
   - 轨迹带发光效果

3. **节点状态动画**
   - hover: 发光边框
   - press: 缩小 + 颜色加深
   - selected: 持续高亮

4. **叠叠乐动画优化**
   - 用 Leafer 的 `Group` 管理节点组
   - 实现更自然的弹跳效果

### 验证标准

- 展开/收起动画流畅（60fps）
- 光标轨迹视觉效果优雅
- 状态动画响应灵敏
- 不影响现有交互逻辑

---

## Phase 3：连线可视化层

**目标**：添加目录连线，增强层级关系可视化。

### 任务清单

1. **基础连线**
   - 父节点到子节点的直线连线
   - 连线颜色同边框色 `rgba(124,58,237,.3)`
   - 连线宽度 1px

2. **流动粒子**
   - 连线上添加流动的小点
   - 粒子从父节点流向子节点
   - 频率可调节（1-2秒一个）

3. **连线高亮**
   - 鼠标悬停节点时，相关连线高亮
   - 高亮颜色 `rgba(0,212,255,.7)`
   - 高亮时粒子加速流动

4. **搜索连线**
   - 搜索匹配时，用连线连接所有匹配节点
   - 形成星座图式的可视化

### 验证标准

- 连线不遮挡节点内容
- 粒子流动流畅
- 高亮效果明显
- 性能影响可控（<5% CPU）

## Phase 4：高级功能层

**目标**：实现设计工具级别的编辑功能。

### 任务清单

1. **缩略图预览**
   - 图片文件显示 Leafer `Image` 缩略图
   - 缩略图尺寸 32x32 或自适应

2. **拖拽重构**
   - 拖起节点放大 + 发光
   - 移动路径带轨迹
   - 放下目标位置预览虚影
   - 实际移动文件（调用 API）

3. **多选框选**
   - 拖拽矩形框选多个节点
   - 框选时节点被圈起
   - 支持批量操作

4. **层级视觉**
   - 第 N 层节点大小 = baseSize * 0.9^N
   - 第 N 层背景透明度 = 0.3 * 0.8^N
   - 形成透视深度感

5. **状态动画**
   - loading: 旋转圆圈
   - error: 红色脉冲
   - success: 绿色光晕一闪

### 验证标准

- 缩略图清晰且不影响性能
- 拖拽手感自然
- 多选操作准确
- 层级视觉明显
- 状态动画响应灵敏

---

## Phase 5：性能优化层

**目标**：处理极端场景，确保海量文件流畅。

### 任务清单

1. **虚拟渲染**
   - 只渲染可视区域 + 缓冲区的节点
   - 滚动时动态加载/卸载

2. **分层渲染**
   - 静态层：未变化的节点
   - 动态层：动画中的节点
   - 交互层：hover/selected 节点

3. **节点缓存**
   - 已渲染节点缓存为图像
   - 减少重复绘制

4. **Web Worker**
   - Leafer 支持 Web Worker 渲染
   - 解放主线程

### 验证标准

- 1000+ 文件滚动流畅（60fps）
- 展开大目录无卡顿
- 内存占用可控（<500MB）

## 技术架构

### 文件结构

```
src/client/modules/
├── tree.ts          → 保留（数据层）
├── tree-leafer.ts   → 新增（Leafer渲染层）
├── tree-animations.ts → 新增（动画定义）
├── tree-connectors.ts → 新增（连线系统）
├── tree-editor.ts   → 新增（编辑功能）
├── ui.ts            → 保留（光标逻辑）
├── orb.ts           → 保留（光球面板）
└── gestures.ts      → 保留（手势）
```

### 渲染流程

```
数据层 (tree.ts)
    ↓ 提供文件数据
渲染层 (tree-leafer.ts)
    ↓ 渲染 Leafer 元素
动画层 (tree-animations.ts)
    ↓ 添加动画效果
连线层 (tree-connectors.ts)
    ↓ 绘制连线
交互层 (gestures.ts)
    ↓ 处理用户输入
```

### 样式继承

```css
/* Leafer 元素样式映射 */
Leafer Rect {
  fill: rgba(18,18,26,.85)  → 对应 var(--surface)
  stroke: rgba(124,58,237,.3) → 对应 border-left 颜色
  strokeWidth: { left: 3, others: 1 }
}

Leafer Text {
  fill: #e0e0e0 → 对应 var(--text)
  fontSize: 12px
  fontFamily: -apple-system, sans-serif
}
```
