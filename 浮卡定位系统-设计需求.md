# 浮卡定位系统 — 设计需求文档

## 1. 背景

当前浮卡从卡片堆发射后，所有卡片定位到屏幕正中央，彼此重叠。需要改为：
- 沿 45° 平行斜线从左下到右上逐张散布
- 避开卡片堆占据的右上角区域（L 形安全区）
- 利用浮卡 z-index 低于卡片堆的特性，不做额外遮盖处理

## 2. 约束条件

### 2.1 层级关系
- 浮卡 z-index：`Z_FLOATING_BASE + index`（50+）
- 卡片堆 z-index：`Z_STACK_BASE + index`（150+）
- 浮卡在卡片堆之下，被堆遮住是天生的层级行为，无需额外处理

### 2.2 卡片尺寸
- 浮卡宽 × 高：`155px × 68px`
- 右下角光球延伸出卡片边缘约 13px（由 `cornerOff = -13`、`cornerSize = 26` 决定）
- 卡片堆卡片：`155px` 宽 + 左右 padding（12px × 2）+ 左右 border（3px + 1px）= `offsetWidth ~183px`

### 2.3 安全边界
- 左边界 `safeL`：若侧栏（sidebar）打开，取 `sidebar.getBoundingClientRect().right + 16`；否则取 16
- 上边界 `safeT`：`16 + 24 = 40`
- 下边界 `safeB`：若主光球（`.light-orb`）存在且其底部 > viewport 高度的 30%，则取 `orb.top - 16` 与 `vh - 16` 的较小者；否则取 `vh - 16`
- 右边界 `fullR`：`vw - 16`

## 3. 安全区模型（倒 L 形）

卡片堆（`.stack-card` 元素集合）占据右上角一个矩形区域，其包围盒记作 `(sL, sR, sT, sB)`。

可用区域呈**倒 L 形**：
- **卡片堆左侧**（`y` 在堆的 y 范围内，`x` 靠左）：允许放置，但卡的右下角光球不能越过堆的左边界
- **卡片堆下方**（`y > sB`）：全宽可用
- **卡片堆上方**（`y < sT`）：**不可用**（修正项）

### AABB 交集检测

判定一个位置 `(x, y)` 是否与卡片堆重叠：

```
cardR = x + FLOATING_CARD_W + ORB_EXT   // 光球延伸后的右边界
cardB = y + FLOATING_CARD_H + ORB_EXT   // 光球延伸后的下边界
// AABB 交集检测：card-bbox 与 stack-bbox 相交则跳过
overlaps = (x < sR && cardR > sL && y < sB && cardB > sT)
```

加上"上侧不可用"的修正，调用 `_calcStackSafeRegion` 前还需额外判断：

```
// 如果 y + FLOATING_CARD_H <= sT，说明整张卡在卡片堆上方 → 跳过该位置
```

## 4. 定位算法

### 4.1 45° 平行斜线模型

扫描线：`x + y = b`（常量和斜线）

- 线间距 `DB = FLOATING_STACK_OFF × 2 = 40px`
- 同一条线上步长：`FLOATING_STACK_OFF = 20px`
- 起始和：`b0 = safeL + safeT + DB`
- 最大和：`bMax = fullR + safeB`

对每条线 `b`，y 范围：`ys = max(safeT, b - fullR)` ~ `ye = min(safeB, b - safeL)`

沿 y 从 `ys` 到 `ye` 步进 `FLOATING_STACK_OFF` 计算 `x = b - y`：

```
for b in range(b0, bMax, DB):
    for y in range(ys, ye, FLOATING_STACK_OFF):
        x = b - y
        if position (x, y) is valid:
            assign to the ci-th floating card
```

### 4.2 位置有效性判定

一个位置 `(x, y)` 有效需同时满足：
1. `x >= safeL` 且 `x + FLOATING_CARD_W <= fullR`
2. `y >= safeT` 且 `y + FLOATING_CARD_H <= safeB`
3. `y + FLOATING_CARD_H > sT`（卡片不在堆的上方）
4. 不与卡片堆 AABB 相交（见第 3 节）

### 4.3 环绕（wrap-around）

当所有已分配位置全部跳过当前卡（即全部位置已用完），取 `ci % totalValidPositions` 重新分配。

### 4.4 兜底

若无任何有效位置，兜底到屏幕居中：`( (vw - 155) / 2, (vh - 68) / 2 )`

## 5. 日志输出

通过 `debugLog()` 将以下数据输出到调试面板，用于验证：
- 安全区边界：`FLOAT L={safeL} T={safeT} B={safeB} fR={fullR} sL={sL} sR={sR} sT={sT} sB={sB}`
- 每次分配：`FLOAT c={index} b={b} y={y} {x},{y}`
- 环绕分配：`FLOAT wrap c={index} …`
- 兜底：`FLOAT fallback {cx},{cy}`
- 发射：`FLOAT launch {focusIndex} {targetLeft},{targetTop}`

## 6. 新增常量与函数

### 6.1 常量
```typescript
const FLOATING_STACK_OFF = 20;
const FLOATING_ORB_EXT = 13;
```

### 6.2 函数签名
```typescript
function _calcStackSafeRegion(): { left: number; top: number }
```
- 查询 DOM 获取侧栏、卡片堆包围盒、主光球位置
- 确保在其他 agent 启动之前不包含对 `_floatingCards` 的直接索引调用（由调用侧传入或通过模块级变量读取）

## 7. 调用点修改

将 `launchFocusedCard()` 中的定位逻辑从：
```typescript
// 旧：硬编码居中
const targetLeft = Math.round((window.innerWidth - FLOATING_CARD_W) / 2);
const targetTop = Math.round((window.innerHeight - FLOATING_CARD_H) / 2);
```

改为：
```typescript
// 新：调用安全区分配器
const stackSafe = _calcStackSafeRegion();
const targetLeft = stackSafe.left;
const targetTop = stackSafe.top;
```

注意 `_floatingCards.push(item)` 在 `_calcStackSafeRegion()` 之后执行，以确保 `_floatingCards.length` 反映当前卡之前的卡数量。

## 8. 验证方式

1. 构建通过：`npm run build`
2. 窗口宽度约 412px（最窄可用宽度），召唤 5 张浮卡，每张应沿 45° 斜线散布
3. 卡片堆展开时召唤浮卡，浮卡不应出现在卡片堆上方
4. 浮卡的右下角光球不被卡片堆遮盖
5. 位置用尽后，新卡调回第一个有效位置（wrap-around）
6. 调试面板日志完整
