> 写于 2026-05-27，合并自 `docs/archive/REFACTOR_THESIS_FULL.md`（愿景蓝图）和 `docs/CARD_SYSTEM_DESIGN.md`（卡片系统设计）。

## 一、核心理念

### 本质

**AI 人机交互的个人工作台，创意与能力的延伸工具。**

KFM 不是一个"文件管理器"，也不是一个"AI 聊天客户端"。它是你在手机上的私人工作台——文件浏览、AI 对话、代码编辑、终端操作，都以"卡片"的形式在工作台上浮动，你随时展开收起、随手调配。

### 一切皆卡片

所有浮动元素都是卡片。AI 对话卡片、调试面板卡片、文件编辑器卡片、工具面板卡片——**唯一的区别是内容**。同一个交互框架支撑所有场景：

- 相同的拖拽行为
- 相同的展开/收起状态机
- 相同的标题栏 + 内容区结构
- 相同的毛玻璃 + 渐变边框视觉

### 用户和 AI 对称操作

这组设计的深层逻辑：**用户能做什么，AI 就能做什么**。

- 用户手指拖拽卡片 → 直接操作 UI
- AI 通过统一协议调用卡片 → 触发相同的行为

能力不是预注册的"AI 工具集"，而是跟着卡片走的。创建一张新卡片的同时，用户和 AI 都获得了新能力。不需要单独注册——每张卡片就是 AI 的一个工具。

### 标记系统（为手机屏幕设计）

手机屏幕小，用户不能像桌面端那样"展示"多个窗口给 AI 看。所以用"标记"来告诉 AI 上下文——反而比桌面更精确，因为必须显式标记：

| 模式 | 数量 | 语义 | AI 收到的信息 |
|------|------|------|-------------|
| 点击光标 | 1 个 | "操作目标在这里" | AI 知道要修改的目标位置 |
| 路标标记 | 多个 | "参考这些内容" | AI 知道用户引用了哪些上下文 |

### "没有主页面"——只有当前卡片

当前布局不是妥协，而是为卡片化做的空间预留：

```
底部:   全局输入栏（常驻入口，随时呼叫 AI）
中央:   工作台网格（放卡片的空间）
侧栏:   收起（需要时唤出 = 文件树卡片展开）
光球:   收起（需要时展开 = AI 对话卡片）
```

不存在"主页面"的概念，只有"当前操作的卡片"——打开侧栏等于展开了文件树卡片，展开光球等于展开了对话卡片，收起全部等于所有卡片最小化，留出空白工作台。

## 二、交互模型

### 卡片生命周期

```
  工具坞 ---拖出-->  浮动卡片  <---左栏点击（文件卡）
     ^                  |
     |                  | 点击标题栏
     |                  v
     +--拖回/关闭--- 展开面板
                       |
                       | 长按
                       v
                    编辑大小
```

### 三种卡片状态

| 状态 | 说明 | 视觉 |
|------|------|------|
| `dock` | 停靠在右侧工具坞 | 仅显示彩色圆点 + 图标 |
| `float` | 在工作台上浮动 | 完整卡片（标题栏 + 内容区） |
| `editing` | 长按进入调整大小 | 浮动卡片 + 加亮阴影 + 拖动改大小 |

### 侧面堆叠卡片的唤出协议

卡片堆不是工具坞——它是"快速选择器"。从右边缘左滑唤出，上下滑切换聚焦，左滑发射为浮动卡片：

```
右边缘 30px 内水平左滑    → 呼出堆叠卡片面板
卡片区域垂直滑动          → 切换聚焦卡片
聚焦卡片再左滑            → 飞入工作台（成为浮动卡片）
右滑（或点击背景）        → 关闭堆叠面板
```

手势判定区分离：
- 边缘区（屏幕右沿 30px 内）→ 只判水平滑动
- 内区（卡片区域）→ 只判垂直滑动
- 两个区域不混淆

### AI 操作卡片（盒子协议）

AI 通过统一的 Tool Call 操作任何卡片：

```typescript
// AI → 框架
{
  "op": "box",
  "boxId": "terminal-1",
  "action": "exec",
  "params": { "cmd": "npm run build" }
}

// 框架 → 目标卡片执行 → 结果返回 AI
```

每个卡片声明它支持的操作：

```typescript
interface CardAction {
  name: string;           // "exec" / "read" / "write"
  description: string;    // "在终端执行命令"
  params: { name: string; type: string; required: boolean }[];
}
```

卡片未创建 → 用户和 AI 都没有这个能力。卡片已创建 → 两者同时获得。这是"对称操作"的具体实现。

## 三、视觉设计

### 卡片样式

```
  ┌─────────────────────────────────┐
  │  AI 对话          [⚙] [×]      │ ← 24-28px 极简标题栏
  │  █████████████████████████████  │    4px 色条（左边缘，专属色）
  │                                 │    彩色图标
  │   (深色毛玻璃内容区)             │
  │                                 │    背景: rgba(20,16,32,0.92)
  │                                 │    backdrop-filter: blur(16px)
  │                                 │    边框: 1px transparent
  │                                 │    圆角: 12px
  │                                 │    阴影: 同当前 orb 风格
  │                                 │
  │  [输入...              发送]    │
  └─────────────────────────────────┘
```

### 配色系统

沿用卡片堆的双色独立随机机制。每种卡片类型分配 Accent Color，渐变方向固定 135deg（右上→左下）：

| 卡片 | 色值 | 用途 |
|------|------|------|
| AI 对话 | `#7c3aed` 紫 | 全局常驻 |
| 调试面板 | `#00c8a0` 青 | 工具坞 |
| 文件编辑 | `#50A880` 绿 | 文件类型映射 |
| 设置 | `#C88C5A` 琥珀 | 工具坞 |
| 新建文件 | `#5088C8` 蓝 | 工具坞 |
| 笔记 | `#B4AA50` 金 | 工具坞 |
| 插件 | `#9650C8` 紫罗兰 | 工具坞 |
| 统计 | `#6C5CC8` 靛蓝 | 工具坞 |
| 关于 | `#B46478` 玫瑰 | 工具坞 |

- color1（渐变起点）对应：右侧光球、图标背景
- color2（渐变终点）对应：左侧光球、图标数字
- 文字颜色统一白色，不派生自卡片色

### 双层 DOM 结构

```html
<div class="card-shell" style="left:X; top:Y; width:W; height:H">
  <!-- 外层壳：渐变边框（通过 padding 挤出） -->
  <!-- 内层：毛玻璃内容区（flow 布局，不绝对定位） -->
  <div class="card-header" style="border-left: 3px solid {accent}">
    <span class="card-icon">{icon}</span>
    <span class="card-name">{name}</span>
  </div>
  <div class="card-content"><!-- 内容插槽 --></div>
</div>
```

边框和背景**必须正交解耦**：外层壳负责渐变边框，内层独立元素负责毛玻璃。修改背景透明度不丢失边框，修改渐变色不干扰背景。

## 四、卡片类型

### 文件卡

| 属性 | 值 |
|------|-----|
| 触发 | 左栏文件树点击 |
| 初始状态 | 直接 `float`（无 dock 态） |
| 关闭 | 关掉即消失，不入工具坞 |
| 内容 | 文件编辑器/查看器 |
| 配色 | 按文件扩展名映射（复用 `getFileColor`） |

### 工具卡

| 属性 | 值 |
|------|-----|
| 来源 | 右侧工具坞 |
| 初始状态 | `dock` |
| 交互 | 拖出 → `float`，拖回工具坞 → `dock` |
| 内容 | 工具面板（设置、调试、笔记等） |
| 配色 | 固定 Accent Color |

### AI 对话卡（特殊工具卡）

| 属性 | 值 |
|------|-----|
| 初始状态 | `float`（始终可见，不 dock） |
| 特殊性 | 唯一的全局常驻卡片 |
| 复用 | 保留现有聊天排版逻辑（消息气泡、输入栏、文字压缩） |

## 五、架构设计

### 目录结构

```
src/client/cards/
├── card-manager.ts          # 治理中心：注册、调度、生命周期
├── card-shell.ts            # 统一 DOM 渲染（标题栏 + 毛玻璃壳 + 拖动）
├── card-dock.ts             # 右侧工具坞渲染
├── types.ts                 # CardConfig 等类型定义
├── cards/
│   ├── ai-chat-card.ts      # AI 聊天（复用 orb.ts 内容）
│   ├── debug-card.ts        # 调试面板（复用 debug-panel.ts 内容）
│   ├── file-card.ts         # 文件编辑（新实现）
│   └── tool-cards.ts        # 设置/笔记/插件等工具卡片
```

### CardConfig

```typescript
type CardState = 'dock' | 'float' | 'editing';
type CardKind = 'file' | 'tool';

interface CardConfig {
  id: string;
  kind: CardKind;
  icon: string;
  name: string;
  accentColor: string;

  // 几何
  defaultWidth: number;
  defaultHeight: number;

  // 初始状态
  initialState?: CardState;

  // 内容渲染
  render: (shell: CardShell) => void;

  // AI 操作声明（盒子协议）
  actions?: CardAction[];

  // 清理
  destroy?: () => void;

  // 位置持久化
  persistGeometry?: boolean;
}
```

### CardShell 职责

一张浮动卡片的完整交互框架：

- DOM 创建（渐变边框壳 + 标题栏 + 内容插槽）
- 拖拽事件绑定（PointerEvent，统一 mouse + touch）
- 位置钳制（屏幕四边 + 输入栏上边界 → 自动压缩）
- 长按切换编辑模式（600ms 检测）
- 编辑模式缩放手柄行为
- 状态切换（expand / collapse / edit）
- onResize(newWidth, newHeight) 回调给内容层

### CardManager 职责

- 注册/注销卡片配置
- 状态切换（dock → float → dock）
- z-index 管理（最新打开的卡片在最前）
- 位置持久化（localStorage）
- 拖动冲突协调
- AI 卡片操作的路由（Tool Call → 对应卡片）

### 工具坞（Card Dock）

右侧竖排图标，每个对应一种工具卡片：

```
  AI     <- 紫色圆点 + 图标，点击切换 float/dock
  调试   <- 青色
  设置   <- 琥珀
  笔记   <- 金色
  插件   <- 紫罗兰
  统计   <- 靛蓝
  关于   <- 玫瑰
```

交互规则：
- 点击 dock 图标 → 卡片从右侧滑入工作台
- 拖动 dock 图标 → 卡片跟随手指拖出
- 卡片 close 时 → 飞回 dock 位置
- AI 卡片特殊：始终 float，点击 dock 图标可 focus/隐藏

### 现有代码复用策略

| 来源 | 去向 | 说明 |
|------|------|------|
| `orb.ts` 拖动/编辑/边界约束 | `card-shell.ts` | 提取为公共函数 |
| `orb.ts` 聊天排版 | `ai-chat-card.ts` | 原地保留 |
| `debug-panel.ts` 拖动 | `card-shell.ts` | 统一路径 |
| `debug-panel.ts` 日志渲染 | `debug-card.ts` | 原地保留 |
| `card-stack.ts` 浮卡框架 | `card-shell.ts` | 统一交互模式 |

## 六、标记系统

用户不需要"展示"上下文给 AI，只需"标记"它。这在手机屏幕上尤为必要。

### 操作流示例

```
1. 在编辑器里点了一下第 42 行       → 光标在那（操作目标）
2. 切换到文件树，点了一个对比文件   → 文件路径被标记（参考来源）
3. 切换到终端，双击某行错误输出    → 错误消息被标记（参考来源）
4. 回到底部输入栏说：
   "根据这个错误，参考 compare.ts 的写法，在这里改一下"

AI 收到的上下文：
  - 活跃卡片：editor-1，第 42 行
  - 标记参考：compare.ts 路径、终端错误行
  → AI 准确理解用户的组合意图
```

### 两种标记

| 模式 | 数量 | 语义 | 实现方式 |
|------|------|------|---------|
| 点击光标 | 1 个 | "操作目标在这里" | 文件树中已实现，编辑器卡片中待实现 |
| 路标标记 | 多个 | "参考这些内容" | 待实现 |

## 七、当前技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 语言 | TypeScript 6 + ES2022 | 全栈 ESM |
| Canvas 渲染 | 自研引擎（Box → Renderer） | 不依赖第三方 Canvas 框架 |
| 动画 | GSAP 3.15 | `animation-registry.ts` 隔离层 |
| 文本测量 | `@chenglou/pretext` | 零 reflow 离屏测量 |
| 布局 | 自研 flex.ts | 轻量 Flexbox 计算 |
| 服务端 | Express 4 | 文件读写 API，走 nginx 反代 |
| 构建 | esbuild | TypeScript → bundle |
